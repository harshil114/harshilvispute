How JavaScript Code Execute

As of we know that when JavaScript executes it's code then it creates the Global Execution Context inside the JavaScript engine. Right? Yes. That code divides into 2 phases in the GEC which is Memory Component & Code Component. So now let's have an example for it.

When this code will executes it's GEC is created that all we know now. This Execution Context created in the 2 phases:   1.  Memory Creation Phase   and   2.  Code Execution Phase

Phase 1 :

In this pahse first JavaScript will allocate memory to the Functions and Variables in the Global Context.

Here, n will store as undefined untill the code doesn't executes and in the space of function it will store whole function which means full code of function get stores here. For square2 and square3 it also stores undefined because they are variables.

So now, phase 1 is completed. JavaScript have assigned values to the variables and functions. Now JavaScript will move to phase 2 which is code execution.

Phase 2 :

In this phase JavaScript go through the code again line by line and executes the code. This is the part where all the functions and calculation are done. By executing code JavaScript will find that n = 2 then it will place that value 2 in the place of undefined in the Global execution.

After this there is nothing to do with line number 2 to 5 because it contains function which is already declared in phase 1.

From here JavaScript moves on line number 6. This line invokes the function. Function invocation is just like a mini program and when this mini program invokes a new Execution context get create in the Code Execution. This Execution Context contains same 2 components which is memory and code.
